Паттерн "Посредник" (или "Mediator") — это поведенческий шаблон проектирования, который позволяет снизить связь между объектами, определив объект-посредник, который инкапсулирует взаимодействие между ними. Вместо того чтобы объекты общались друг с другом напрямую, они передают сообщения посреднику, который решает, кому эти сообщения передать дальше.
Структура паттерна "Посредник":
1.	Colleague (Коллега):
o	Абстрактный класс или интерфейс, определяющий общий метод общения с посредником.
2.	ConcreteColleague (Конкретный коллега):
o	Конкретные классы, реализующие поведение коллег, участвующих в общении через посредника.
3.	Mediator (Посредник):
o	Интерфейс или абстрактный класс, определяющий методы для связи с коллегами.
4.	ConcreteMediator (Конкретный посредник):
o	Реализует конкретное поведение посредника, управляющее взаимодействием между коллегами.


Паттерн "Посредник" полезен в ситуациях, когда необходимо минимизировать прямую зависимость между объектами системы,
обеспечив централизованный контроль над их взаимодействием.
